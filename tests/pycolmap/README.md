# pycolmap round-trip tests

Validates that exports produced by `src/parsers/writers.ts` load cleanly in the
upstream `pycolmap` package — the ground truth for the COLMAP binary format.

## What's covered

| Scenario | Guards the fix for |
|---|---|
| `untransformed` | baseline — writer output is self-consistent |
| `transform_baked` | `applyTransformToData` realizes `points2D` before dropping WASM |
| `deletions_applied` | `applyDeletionsToData` in WASM mode builds+filters `points3D` tracks |
| `bug_empty_points2D` | negative control — the pre-fix state produces dangling track refs |

Each scenario is a COLMAP `sparse/0/` directory containing `cameras.bin`,
`images.bin`, `points3D.bin`.

## Running

Requires `uv` (Python) and the bicycle dataset at
`../../../360_v2/bicycle/sparse/0/`.

```bash
cd tests/pycolmap

# First run — installs pycolmap via uv and generates fixtures
uv run pytest

# Force-regenerate fixtures before testing
REGEN_FIXTURES=1 uv run pytest
```

Fixtures are generated by
`npx vite-node tests/pycolmap/generate_fixtures.ts` (invoked automatically
by the pytest session fixture if fixtures are missing).

If the bicycle dataset isn't present, the tests are skipped cleanly.

## Why vite-node

The fixture generator imports real app code (writers, transform math,
deletion filter) that pulls in the Vite module graph via TypeScript.
`vite-node` resolves those imports the same way the dev server does; plain
`tsx` chokes on things like the `libarchive.js` default export used
transitively by the deletion action.
