# COLMAP 4.1 Gap-Closes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the two residual gaps from the COLMAP-4.1 distortion-hardening effort — (A) the self-referential-test risk for EUCM / DIVISION / the fisheye sub-models, and (B) the WASM C++ parser rejecting the new models (id > 10) plus the absence of any TS↔C++ contract.

**Architecture:** Two independent efforts on one branch off `main` (`feature/colmap-4.1-gap-closes`, off `8bd48a2`). Effort A is test-only (adds *independent, COLMAP-derived* oracles that would catch the class of bug the RAD_TAN Critical belonged to); it changes production code only if an oracle exposes a real deviation. Effort B edits the git-ignored WASM C++ source, rebuilds via the bundled emsdk, commits the rebuilt `public/wasm/` artifacts, and adds a runtime parity test that pins the committed binary's `getNumCameraParams` to the registry.

**Tech Stack:** TypeScript + Vitest (jsdom). Emscripten (bundled `colmap-wasm/emsdk/`, v4.0.23) + bundled CMake 3.31.4 for the C++→WASM rebuild. COLMAP `src/colmap/sensor/models.h` is the math source-of-truth.

## Global Constraints

- **The oracle must be independent.** Every new oracle value is hand-computed from COLMAP's algebraic formula (transcribed into a test comment), NOT produced by calling our implementation. An oracle that reads back our own output proves nothing — that blind spot hid the RAD_TAN Critical. Each oracle must be demonstrably non-tautological: it would fail if the formula were perturbed.
- **Registry is the single source of truth for param counts.** `getCameraModelNumParams(id)` (`src/utils/cameraModelRegistry.ts`) is authoritative. C++ `GetNumParams` must match it exactly for all 18 models.
- **`colmap-wasm/` is git-ignored** ("only built assets in `public/wasm/` are committed"). C++ edits are local-only; the *committed deliverable* of Effort B is the rebuilt `public/wasm/colmap_wasm.{js,wasm}`. Any test must exercise the committed binary or the registry — never read the git-ignored C++ source (absent in CI).
- **Do not weaken existing tests.** The `undistortionParity.test.ts` contract (structural exhaustiveness + non-tautological CPU↔GLSL parity) and the full suite (2672 green at branch base) must stay green.
- **Behavior-preserving for models 0–10.** These already work; no change to their observable output.
- Commit trailer: `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`.

- [ ] **Step 2: Write the EUCM oracle (failing first if impl were wrong).** Hand-compute one asymmetric undistorted→distorted pair from the EUCM forward formula, showing the arithmetic in comments. Worked starting value to VERIFY against source (adjust if COLMAP's convention differs):

```ts
// EUCM forward (undistorted normalized (a,b) with Z=1 → distorted normalized (u,v)):
//   d     = sqrt(beta*(a^2+b^2) + 1)
//   denom = alpha*d + (1 - alpha)
//   u = a/denom,  v = b/denom
// For a=0.3, b=-0.2, alpha=0.6, beta=1.2:
//   a^2+b^2 = 0.13 ;  d = sqrt(1.2*0.13 + 1) = sqrt(1.156) = 1.0751744
//   denom   = 0.6*1.0751744 + 0.4 = 1.0451046
//   u = 0.3/1.0451046 =  0.2870535 ;  v = -0.2/1.0451046 = -0.1913690
const EUCM = { fx:1, fy:1, cx:0, cy:0, alpha:0.6, beta:1.2 };
expect(distortNormalized(CameraModelId.EUCM, { x:0.3, y:-0.2 }, EUCM))
  .toMatchNormalized({ x:0.2870535, y:-0.1913690 }, 1e-6);   // helper or two closeTo asserts
// round-trip (locks inverse with the SAME hand-picked pair):
expect(undistortNormalized(CameraModelId.EUCM, { x:0.2870535, y:-0.1913690 }, EUCM))
  .toMatchNormalized({ x:0.3, y:-0.2 }, 1e-6);
```

- [ ] **Step 3: Write the DIVISION oracle.** The division/undistort direction is the clean closed form — hand-pick a *distorted* point and compute the undistorted independently, then lock both directions:

```ts
// DIVISION undistort (distorted (dx,dy) → undistorted (ux,uy)):  u = d / (1 + k*|d|^2)
// For dx=0.4, dy=0.3, k=0.1:  |d|^2 = 0.25 ; 1 + 0.1*0.25 = 1.025
//   ux = 0.4/1.025 = 0.3902439 ;  uy = 0.3/1.025 = 0.2926829
const DIV = { fx:1, fy:1, cx:0, cy:0, kDiv:0.1 };   // confirm the DIVISION coeff field name in the impl
expect(undistortNormalized(CameraModelId.DIVISION, { x:0.4, y:0.3 }, DIV))
  .toMatchNormalized({ x:0.3902439, y:0.2926829 }, 1e-6);
// forward (distort) closes back to the hand-picked distorted point:
expect(distortNormalized(CameraModelId.DIVISION, { x:0.3902439, y:0.2926829 }, DIV))
  .toMatchNormalized({ x:0.4, y:0.3 }, 1e-6);
```
Also add one SIMPLE_DIVISION case (same math, `f,cx,cy,k` layout) to prove the shared `division` strategy covers both ids.

- [ ] **Step 4: Prove non-tautological.** Temporarily perturb the expected value (e.g. swap `alpha`↔`beta` in your head and confirm the numbers move by ≫ tol; or flip a sign) — document in a comment the delta a wrong formula produces, so a future reader sees the oracle has teeth. - [ ] **Step 5: Run.** `npm run test:run -- src/utils/cameraModelOracles.test.ts` → expect PASS. If EUCM or DIVISION FAILS, our impl deviates from COLMAP — STOP, treat as a Critical: fix `cameraUndistortion.ts` (and the GLSL + parity reference if the same deviation exists there), then re-run.
